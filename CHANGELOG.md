## [Unreleased]

### Added — Metadata plugin refactor (chunks 1–7)

The bundled metadata plugin surface has been ported onto a canonical
manifest + ABI. See `docs/superpowers/plans/2026-04-20-metadata-plugin-refactor.md`
for the full spec and `plugins/README.md` for the end-state matrix.

- **New SDK traits.** `Plugin` (lifecycle — `manifest()`, `init()`,
  `shutdown()`) + `CatalogPlugin` (per-verb: `search` required;
  `lookup` / `enrich` / `get_artwork` / `get_credits` / `related`
  optional with `NOT_IMPLEMENTED` defaults). The legacy `StuiPlugin`
  trait is `#[deprecated]` but retained for non-metadata plugin kinds
  during transition.
- **Init lifecycle.** `Plugin::init(&mut self, &InitContext)` runs once
  after instantiation. `PluginInitError::MissingConfig { fields, hint }`
  routes to `PluginStatus::NeedsConfig` without registering the
  supervisor; `Fatal` and plumbing errors go to `Failed`.
- **Wire-format envelopes.** `InitRequest` / `InitResultEnvelope` cross
  the WASM ABI; the plugin-side `stui_init` export is generated by
  `stui_export_catalog_plugin!`. New per-verb IPC variants
  (`Request::{Lookup, Enrich, GetArtwork, GetCredits, Related}`) expose
  every verb on the daemon socket.
- **CLI tooling.** `stui plugin {init, build, lint, test, install}` (the
  `cli/` crate) scaffolds new plugins, compiles to `wasm32-wasip1`,
  validates manifests against the canonical schema, and dev-symlinks
  into `~/.stui/plugins/<name>/`.
- **Bundled plugins migrated or added:** `tmdb` (reference),
  `omdb` (search + lookup; no enrich/artwork/credits/related per
  upstream), `anilist` (fixed TRENDING pagination + Movie/Series
  format-filter bug; full six verbs), `kitsu` (show_type-filtered
  search + kitsu + myanimelist id lookup), `discogs` (dropped Track
  scope; separated genre vs format/label; api key optional per v2.0),
  `lastfm` (simplified config; fixed listeners-as-genre bug), plus
  the new `musicbrainz` provider (search + lookup + enrich +
  get_artwork + get_credits).
- **Dropped from bundled:** `imdb-provider`, `javdb`, `r18`,
  `listenbrainz-provider`, `subscene`, `kitsunekko`, `yify-subs`,
  `torrentio-rpc` — per spec §5. Non-metadata plugins live in the
  `stui_plugins` repo now.

### Changed

- **WASM host imports pinned to the `stui` module.** The SDK's
  `extern "C"` blocks declaring `stui_log` / `stui_http_get` /
  `stui_http_post` / `stui_cache_*` / `stui_auth_*` / `stui_exec`
  carry `#[link(wasm_import_module = "stui")]`; the runtime's linker
  had always registered them under `"stui"`, but default rustc import
  module is `"env"`, which had silently blocked every plugin from
  instantiating against the new SDK.
- **Plugin singleton uses `Mutex<T>`** instead of `RefCell<T>` inside
  `stui_export_plugin!` / `stui_export_catalog_plugin!`, so the plugin
  crates (workspace members) compile for the host target during
  `cargo check --workspace`.
- **`WasmInner::abi_version` is async.** The store runs with
  `config.async_support(true)`; the previous sync `f.call()` tripped a
  wasmtime trap the first time any plugin instantiated cleanly.

### Added

- **Circuit breakers for providers** - Prevents cascading failures by temporarily disabling providers after consecutive failures
  - New `CircuitBreaker` module in `runtime/src/providers/circuit_breaker.rs`
  - State machine: Closed → Open → Half-Open → Closed
  - Configurable failure threshold (default: 5), recovery timeout (default: 60s)
  - Wired into stream resolution pipeline
- **Plugin resource limits** - CPU and memory limits for plugin processes
  - `max_cpu_percent` via nice/renice on Linux
  - `request_timeout_ms` for individual RPC calls
  - `memory_mb` tracking in watchdog
  - `timeout_count` statistics
- **Watch history migration (Go → Rust)** - Centralized playback position tracking
  - New `watchhistory` module in Rust runtime
  - IPC-backed Go client with local cache
  - Auto-complete threshold (90%) preserved
- **Media cache migration (Go → Rust)** - Centralized catalog caching
  - New `mediacache` module in Rust runtime
  - Runtime caches live grid updates directly
  - IPC methods for Go TUI access
- **Stream policy scoring migration (Go → Rust)** - Policy-based stream ranking
  - `StreamPreferences` and `UserPolicy` in Rust quality module
  - IPC handler for rank operations
  - Replaces duplicated Go scoring logic

### Changed

  - CI uses `-D warnings` to deny all warnings

  - `type-complexity-threshold = 100`
